# Mezzofy Brand Guidelines

## Colors
- Primary Orange: #f97316
- Black: #000000
- White: #ffffff
- Light Orange (alternate rows / backgrounds): #fef3ea

## Typography
- Primary Font: Inter (sans-serif); fallback: Helvetica Neue, Arial
- Heading Weight: 700 (Bold)
- Body Weight: 400 (Regular)
- Monospace Font: JetBrains Mono (code/data); fallback: Courier New

## Voice & Tone
- Confident and authoritative тАФ state facts directly, avoid hedging language
- Friendly and approachable тАФ frame interactions as partnerships, not transactions
- Results-focused тАФ always tie features and data back to business outcomes
- Clear and concise тАФ no jargon, no passive voice, no filler phrases
- Positive but honest тАФ do not oversell; acknowledge limitations plainly

## Document Standards

### All Documents (apply to every format)
- Always include "Generated by Mezzofy AI Assistant" in footer
- Use orange (#f97316) for all primary headings (H1, H2, slide titles)
- Use black (#000000) for body text
- Use white (#ffffff) backgrounds for content areas
- Company name is always "Mezzofy" тАФ never "mezzofy" or "MEZZOFY"
- Date format: DD Month YYYY (e.g., 21 March 2026)

### PowerPoint / PPTX Presentations
- Cover slide: Black background, orange accent bars, white title text
- Content slides: White background, orange top bar (6pt), orange section headings
- Divider / section break slides: Orange background, white title
- Thank You / closing slide: Black background, orange logo area, white text
- Minimum structure: Cover тЖТ Agenda тЖТ [Content Slides] тЖТ Thank You
- Font sizes: Title 36тАУ44pt, Subtitle 24тАУ28pt, Body 18тАУ20pt, Caption 14pt
- Maximum 6 bullet points per slide; prefer visuals over walls of text

#### PPTX QA Checklist (required before delivery)
Run the full visual inspection loop тАФ do not skip even for short decks:

```bash
# 1. Convert to PDF
python scripts/office/soffice.py --headless --convert-to pdf output.pptx

# 2. Render each slide as an image
rm -f slide-*.jpg
pdftoppm -jpeg -r 150 output.pdf slide

# 3. Inspect every slide image with the view tool
ls -1 "$PWD"/slide-*.jpg
```

Inspect each rendered slide and fix before presenting:
- Text overflow or clipping at box boundaries
- Overlapping shapes or text elements
- Low-contrast text (e.g., orange text on white near orange shapes)
- Elements misaligned or outside the 0.5" slide margin
- Logo missing or using the wrong variant (dark vs light)
- Brand colours not applied тАФ headings must be orange (#f97316), body black
- Leftover placeholder text (check with `python -m markitdown output.pptx | grep -iE "lorem|TODO|xxx|\[insert"`)

Re-render and re-inspect after every fix. Do not present until a full pass is clean.

### PDF Reports
- Page header: "Mezzofy" branding top-right in orange on every page
- Page footer: "Generated by Mezzofy AI Assistant | Page N" centered
- Table headers: Orange background (#f97316), white text, bold
- Alternate table rows: Light orange (#fef3ea)
- Charts/graphs: Use orange as primary series color, black for secondary
- Section headings (H1): Orange, 16pt bold; H2: Black, 13pt bold

#### PDF QA Checklist (required before delivery)
PDF output is already in the correct format тАФ render pages directly:

```bash
# Render each page as an image
rm -f page-*.jpg
pdftoppm -jpeg -r 150 output.pdf page

# Inspect every page image with the view tool
ls -1 "$PWD"/page-*.jpg
```

Inspect each rendered page and fix before presenting:
- Text clipped at margins or overflowing content boxes
- Unicode characters rendering as black boxes (never use Unicode subscripts/superscripts in reportlab тАФ use `<sub>` and `<super>` tags instead)
- Table column widths misaligned or headers missing orange background
- Page header or footer absent on any page
- Brand colours not applied тАФ H1 must be orange, body black, table headers orange
- Logo absent or incorrectly sized in page header

Re-render and re-inspect after every fix. Do not present until a full pass is clean.

### Word Documents / DOCX
- Document header: "Mezzofy AI Assistant" right-aligned, orange, 10pt
- Document footer: "Generated by Mezzofy AI Assistant" centered, 9pt grey
- H1: Orange (#f97316), 20pt bold

#### DOCX QA Checklist (required before delivery)
DOCX requires both structural validation and visual inspection:

```bash
# 1. Structural validation
python scripts/office/validate.py output.docx

# 2. Check for leftover placeholder text
python -m markitdown output.docx | grep -iE "lorem|ipsum|TODO|xxx|\[insert"

# 3. Convert to PDF and render pages for visual inspection
python scripts/office/soffice.py --headless --convert-to pdf output.docx
rm -f page-*.jpg
pdftoppm -jpeg -r 150 output.pdf page
ls -1 "$PWD"/page-*.jpg
```

Inspect each rendered page and fix before presenting:
- Validation errors must all be resolved before proceeding
- Leftover placeholder text found by grep must be replaced
- Table cells with black backgrounds (caused by `ShadingType.SOLID` тАФ use `ShadingType.CLEAR`)
- Heading colours not applied тАФ H1 and H2 must be orange (#f97316)
- Table widths using `WidthType.PERCENTAGE` (breaks in Word/Google Docs тАФ use `WidthType.DXA`)
- Missing header or footer on any page
- Tracked changes or comments not authored as "Mezzofy AI Assistant"

Re-render and re-inspect after every fix. Do not present until validation passes and a full visual pass is clean.

### Excel / XLSX Spreadsheets
- Header row: Orange background (#f97316), white bold text
- Alternate data rows: Light orange (#fef3ea) and white
- Total / summary rows: Black background, white bold text
- Column headers: Bold, centered
- Numeric data: Right-aligned; text data: Left-aligned
- Tab/sheet names: Title case, descriptive (e.g., "Monthly Sales", not "Sheet1")

#### XLSX QA Checklist (required before delivery)
XLSX QA is formula-first тАФ visual inspection alone cannot catch calculation errors:

```bash
# 1. Recalculate all formulas and check for errors (mandatory if any formulas present)
python scripts/recalc.py output.xlsx

# 2. If status is "errors_found", fix every listed location and recalculate again
# Common errors:
#   #REF!   тАФ cell reference points to a deleted or invalid cell
#   #DIV/0! тАФ formula divides by zero; add an IFERROR or check denominators
#   #VALUE! тАФ wrong data type fed into a formula
#   #NAME?  тАФ unrecognised function name (typo or missing add-in)
```

Inspect the recalc output and fix before presenting:
- Zero formula errors is a hard requirement тАФ do not present a file with any `#REF!`, `#DIV/0!`, `#VALUE!`, or `#NAME?` errors
- All totals, percentages, and ratios must use Excel formulas, not hardcoded Python values
- Header row must have orange background (#f97316) with white bold text
- Alternate rows must use light orange (#fef3ea) тАФ not plain white throughout
- Summary/total rows must have black background with white bold text
- Sheet tab names must be descriptive Title Case тАФ not "Sheet1", "Sheet2"
- Financial models: blue text for hardcoded inputs, black for formulas, green for cross-sheet links

Do not present until `recalc.py` returns `"status": "success"` with zero errors.

## Output QA Standards

Every document generated by Mezzofy AI Assistant must pass its format-specific QA checklist (see each format section above) before being presented to the user. This is non-negotiable regardless of document length or complexity.

### General Principles

**Assume the first render has problems.** Approach QA as a bug hunt, not a confirmation step. Finding zero issues on first inspection means the inspection was not thorough enough.

**Fix, then re-verify.** One fix often introduces another problem. After any correction, re-run the full QA step for the affected format тАФ do not declare success after fixing without re-checking.

**Never present a file that has not passed QA.** If the QA loop cannot be completed (e.g., a tool is unavailable), state this explicitly and describe what was not verified.

### QA by Format тАФ Quick Reference

| Format | Primary QA method | Hard requirement |
|--------|-------------------|------------------|
| PPTX | Visual: soffice тЖТ pdftoppm тЖТ view each slide | No layout defects, brand colours applied |
| PDF | Visual: pdftoppm тЖТ view each page | No clipping, no broken chars, brand applied |
| DOCX | Structural: validate.py + grep; Visual: soffice тЖТ pdftoppm | Validation clean, no placeholder text |
| XLSX | Formula: recalc.py тЖТ JSON error report | Zero formula errors (`status: success`) |

### Common Issues Caught by QA

These are the most frequent defects found during inspection тАФ check for all of them:

**PPTX / PDF**
- Text overflowing its bounding box (especially slide titles and card bodies)
- Shapes overlapping text or each other due to coordinate errors
- Wrong logo variant тАФ use `logo_light.png` on black/orange, `logo_dark.png` on white
- Missing brand colours тАФ orange headings replaced by default black or blue

**DOCX**
- `ShadingType.SOLID` on table cells rendering as solid black тАФ always use `ShadingType.CLEAR`
- `WidthType.PERCENTAGE` on tables breaking layout in Word and Google Docs тАФ use `WidthType.DXA`
- Unicode bullets (`тАв`) instead of `LevelFormat.BULLET` numbering config causing double bullets

**XLSX**
- Python-calculated values hardcoded into cells instead of Excel formulas тАФ cells will not update when source data changes
- Formulas written but not recalculated тАФ openpyxl stores formula strings; `recalc.py` is required to compute values
- `data_only=True` used when saving тАФ permanently replaces all formulas with static values

- Do not include clipart or low-resolution images
- Do not use Comic Sans, Times New Roman, or decorative fonts
