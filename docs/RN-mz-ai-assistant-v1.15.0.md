## Summary

This release delivers two improvements:

1. **Image Analysis on Upload** ‚Äî The `/files/upload` endpoint now performs inline OCR and vision
   analysis on image files. When a user uploads an image, the server extracts text (OCR) and
   generates a visual description via Claude. The `image_analysis` field is returned in the
   upload response and used by the chat handler to route images through the vision pipeline.

2. **Remove Audio/Video Input Modes** ‚Äî The mobile chat grid previously offered 9 input modes
   including Video and Audio. These two modes are removed (grid is now 7 modes), simplifying
   the UX and eliminating functionality that was not production-ready.

---

## New Features

### Feature 1 ‚Äî Image OCR + Vision Analysis on File Upload

**Backend (`server/app/api/files.py`):**

When a file with an image extension (`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.bmp`) is
uploaded via `/files/upload`, the server now:

1. Reads the uploaded bytes
2. Calls the Claude vision API (base64-encoded image content) to extract OCR text and generate
   a visual description
3. Returns `image_analysis: { ocr_text, description, model }` in the upload response alongside
   the existing `file_id`, `filename`, and `content_type` fields

**Backend (`server/app/api/chat.py`):**

The `send_artifact()` function now detects image extensions via the `_IMAGE_EXTENSIONS` set
and routes image artifacts through `image_handler` (vision pipeline) instead of the generic
document handler.

**Tests (`server/tests/test_files.py`):**

5 new tests added covering: image upload with analysis, non-image upload (no analysis field),
unsupported format rejection, oversized file rejection, and analysis failure fallback.

---

### Feature 2 ‚Äî Remove Video and Audio Input Modes from Chat Grid

**Mobile (`APP/src/utils/theme.ts`):**

`INPUT_MODES` array reduced from 9 entries to 7. The `video` and `audio` mode entries are
removed. All remaining modes (document, image, camera, text, voice, microphone, location)
are unaffected.

**Mobile (`APP/src/screens/ChatScreen.tsx`):**

`handleModeAction()` simplified ‚Äî the `video` and `audio` cases are removed from the switch
statement. The `MediaType` import (used only by the removed video mode) is also removed,
eliminating an unused import warning.
